Recommendation: Add a second NVMe SSD — 2 TB minimum, 4 TB strongly preferred (e.g., Samsung 990 Pro 4TB, WD Black SN850X 4TB). On SNO, the LVM Storage operator pools all local disks together automatically. Your existing 1 TB stays for RHCOS/OCP, and the new disk becomes the LVM pool for PVCs.

Network Isolation

MAS uses NetworkPolicy resources to enforce a default deny-all policy in each namespace. Only traffic that MAS explicitly allows can flow between namespaces. You cannot customize these policies — they are managed by the MAS operator. To see them: oc -n mas-<instance>-core get networkpolicies

Administrative Roles

Security groups: Users are added to security groups. Groups define which apps and what permissions. A user's entitlement tier is automatically set to the highest tier required by all their combined security groups. Default groups exist for each app (e.g. MONITOR_ADMIN, MONITOR_USERS, MVI_ADMIN, MVI_USERS, ASSISTADMIN, ASSISTTECHUSERS).

Load Balancing on SNO — MetalLB Replaces External Hardware LB

PurposePortHow on SNO
OCP API server6443 (TCP)SNO node IP directly — no LB needed, single node has fixed IP
OCP app traffic (MAS)443 (HTTPS) + 80 (HTTP)MetalLB operator assigns a VIP from your local subnet
DNS on SNO: Point all ~20 MAS DNS records to your machine's static IP (or MetalLB VIP for app traffic). Use dnsmasq on your bastion or router for local DNS — add address=/.mas.local/<your-machine-ip> to catch all subdomains in one line.
