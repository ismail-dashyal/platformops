Master Checklist: Single Node OpenShift (SNO)

Here is your definitive, master checklist for installing a Single Node OpenShift (SNO) bare-metal cluster on your Dell OptiPlex 5080.

If you follow these instructions precisely, you will not need any outside documentation.

Phase 1: Hardware Preparation (The Wipe)

What to do:
Prepare the Dell OptiPlex 5080 for total data destruction. You explicitly do not want backups, so we will allow the installer to completely overwrite the disk.
Where to do it:
Physically on the Dell OptiPlex 5080.
How to do it:
  1. Unplug any external USB drives or secondary hard drives attached to the OptiPlex.
  2. Leave only the target 477 GB SSD installed inside the machine.
What to keep in mind: You do not need to manually format the drive beforehand. The Red Hat CoreOS (RHCOS) installer will automatically wipe the partitions, destroy Windows and Kali, and format the drive for OpenShift during Phase 6.

Phase 2: Gather Prerequisites

What to do:
Collect the necessary files, credentials, and physical media required for the installation.
Where to do it:
On your separate Linux machine (the "Bastion Host") and a web browser.
How to do it:
  1. USB Drive: Locate an 8 GB or larger USB flash drive. Plug it into your Bastion Host.
  2. Red Hat Account: Create or log into a free developer account at console.redhat.com.
  3. Pull Secret: Navigate to OpenShift > Downloads in the Red Hat console. Click Download pull secret and save the pull-secret.json file to your Bastion Host.
  4. SSH Key: Open a terminal on your Bastion Host and type ssh-keygen -t ed25519 -C "mas-admin". Press Enter through all prompts to accept the default file location (~/.ssh/id_ed25519) and leave the passphrase empty.
  5. Retrieve Public Key: Type cat ~/.ssh/id_ed25519.pub and copy the output. You will need this later.
What to keep in mind: An empty passphrase on your SSH key is acceptable for this setup, as it allows seamless automation and access if you need to troubleshoot the node later.

Phase 3: Configure Network & DNS (The Critical Gateway)

What to do:
Assign a static IP to the OptiPlex and route DNS traffic to it.
Where to do it:
Your network router and your local DNS server (e.g., dnsmasq on your Bastion Host).
How to do it:
  1. Static IP: Log into your router, find the MAC address of the OptiPlex, and assign it a permanent IP (e.g., 192.168.1.50).
  2. DNS Records: On your Bastion Host (assuming dnsmasq is installed), edit /etc/dnsmasq.conf and add:
    • address=/api.sno.mas.local/192.168.1.50
    • address=/api-int.sno.mas.local/192.168.1.50
    • address=/.apps.sno.mas.local/192.168.1.50
  3. Restart DNS: Run sudo systemctl restart dnsmasq.
  4. Verify: Run ping api.sno.mas.local. It must return 192.168.1.50.
What to keep in mind: If ping fails, STOP. Do not proceed to Phase 4 until your DNS correctly resolves to the OptiPlex's IP.

Phase 4: Prepare the Bastion Host Tools

What to do:
Install the command-line tools you need to interact with the cluster once it is built.
Where to do it:
The terminal of your Bastion Host.
How to do it:
  1. Go to console.redhat.com/openshift/downloads.
  2. Under "OpenShift Command Line Interface (oc)", download the Linux binary.
  3. Extract the file: tar -xvf openshift-client-linux.tar.gz.
  4. Move the binaries to your system path: sudo mv oc kubectl /usr/local/bin/.
  5. Verify installation: oc version.
What to keep in mind: The oc tool is the OpenShift equivalent of kubectl. You will use it for everything post-installation.

Phase 5: Generate and Flash the Installer ISO

What to do:
Define your cluster in the Red Hat portal and create the bootable installation media.
Where to do it:
Web browser (console.redhat.com) and your Bastion Host.
How to do it:
  1. Log into console.redhat.com/openshift, click Create cluster, go to the Datacenter tab, and select Assisted Installer.
  2. Cluster Details: Name the cluster sno. Set the base domain to mas.local. Select x86_64. Check the box for Install single node OpenShift (SNO).
  3. Operators: Skip adding additional operators (like OpenShift Virtualization) for now to save resources. Click Next.
  4. Host Discovery: Paste the contents of your pull-secret.json into the pull secret box. Paste your SSH public key (id_ed25519.pub) into the SSH box.
  5. Generate ISO: Click Generate Discovery ISO. Select Full image file. Click Generate and download the ISO to your Bastion Host.
  6. Flash the USB: Find your USB drive identifier (e.g., /dev/sdb or /dev/sdc) using the lsblk command. Run sudo dd if=path/to/downloaded.iso of=/dev/sdX bs=4M status=progress (Replace sdX with your actual USB drive letter).
What to keep in mind: Triple-check your dd command target. Choosing the wrong /dev/sdX will overwrite your Bastion Host's operating system.

Phase 6: Boot and Install

What to do:
Boot the OptiPlex using the USB drive and initiate the automated installation.
Where to do it:
Physically on the OptiPlex, then back to the Assisted Installer web browser.
How to do it:
  1. Insert the flashed USB drive into the OptiPlex. Power it on and press F12 (or F2) repeatedly to enter the boot menu.
  2. Select the USB drive to boot. The screen will show text scrolling as RHCOS Live boots up. Leave the machine alone.
  3. Go back to the Assisted Installer web page on your Bastion Host. Within 5–10 minutes, your OptiPlex will appear in the web interface as a discovered host.
  4. Verify the IP address matches your static IP. Change the hostname to sno-master (or whatever you prefer).
  5. Click Next to reach the Storage page. Select your 477 GB SSD as the installation disk. Click Next.
  6. Review the summary page. All pre-installation checks should be green. Click Install cluster.
What to keep in mind: The OptiPlex will reboot several times. The web console progress bar will track everything. Do not touch the OptiPlex, unplug the network, or remove the USB until the web console explicitly says the installation is 100% complete. This takes 45 to 60 minutes.

Phase 7: Verification and Hand-off

What to do:
Confirm the cluster is alive and authenticate as the administrator.
Where to do it:
The terminal of your Bastion Host.
How to do it:
  1. Once the web console says "Cluster installed successfully," it will provide a link to download your kubeconfig file and display a kubeadmin username and password.
  2. Download the kubeconfig file to your Bastion Host.
  3. In your terminal, point your oc tool to that file by running: export KUBECONFIG=/path/to/your/downloaded/kubeconfig.
  4. Run oc get nodes.
  5. Run oc get clusteroperators.
What to keep in mind: When you run oc get nodes, your single node must show a status of Ready. When you run oc get clusteroperators, all operators should eventually show AVAILABLE: True, PROGRESSING: False, and DEGRADED: False. If they are still progressing, simply wait another 10 minutes. Your cluster is now fully operational.
